# 코드 단순화 (Code Simplifier)

최근 변경된 파일들을 분석하여 불필요한 복잡성을 제거합니다.

## 1단계: 변경된 파일 파악

```bash
git diff --name-only HEAD~1 2>/dev/null || git diff --name-only --cached
```

## 2단계: 각 파일을 읽고 다음 관점에서 분석

### 제거 대상
- **불필요한 추상화**: 한 번만 사용되는 헬퍼/유틸리티 함수
- **과도한 래핑**: 단순히 다른 함수를 감싸기만 하는 함수
- **중복 코드**: 같은 로직이 반복되는 부분
- **죽은 코드**: 사용되지 않는 import, 변수, 함수
- **불필요한 타입**: `any` 사용, 과도하게 복잡한 제네릭
- **과도한 에러 핸들링**: 발생할 수 없는 에러에 대한 방어 코드
- **불필요한 주석**: 코드 자체로 명확한 부분의 주석

### 단순화 원칙
1. 3줄 이하의 유사 코드는 추상화하지 않는다
2. 한 번만 사용되는 함수는 인라인한다
3. 설정 가능성(configurability)보다 직접 구현을 선호한다
4. 미래 요구사항이 아닌 현재 요구사항에 맞춘다

## 3단계: 변경 적용

- 각 단순화를 설명하고 적용합니다
- 변경 후 타입 체크와 테스트가 통과하는지 확인합니다

```bash
cd frontend && pnpm type-check 2>&1 | tail -5
cd backend && uv run pytest --tb=short -q 2>&1 | tail -10
```

## 4단계: 결과 요약

```
## 단순화 결과

### 변경 파일
- `파일명`: 변경 내용 설명

### 제거된 항목
- 제거된 코드 라인 수
- 제거된 함수/클래스 수

### 검증
- Type Check: ✅/❌
- Tests: ✅/❌
```
